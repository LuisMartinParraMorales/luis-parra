# Public Profile Draft

## Short version

Luis Parra is a software and ML engineer who turns transport, geospatial, and simulation workflows into dependable decision-support products.

## Medium version

I work where analytical logic meets software delivery. The problems I enjoy most usually start in an awkward state: a spreadsheet workflow, a simulation output, a GTFS pipeline, a prototype notebook, or a tool that only analysts can operate confidently. My best work is taking that underlying logic seriously and turning it into something cleaner to run, easier to explain, and safer to extend.

Across recent projects, that has meant building graph-based transit tooling, workbook-driven prioritisation services, map-heavy analytical interfaces, journey-time runtime components, and simulation post-processing pipelines. I am especially interested in work where algorithms, cloud/runtime thinking, and product usability all matter at the same time.

## Long version

I am most useful when the technical challenge is not only "build the model" or "ship the UI," but connect the whole system properly. A recurring pattern in my work is moving from prototype to product: keeping the analytical intent, making the contract clearer, tightening the validation story, and giving users an interface or workflow they can actually trust.

That pattern shows up in different forms:

- GTFS and generalized-cost graph workflows for transport accessibility and skimming
- internal prioritisation tools that moved from workbook-heavy prototypes toward cleaner backend and UI boundaries
- journey-time runtime and validation workflows for larger transport analytics platforms
- simulation post-processing, intervention analysis, and map/report outputs for agent-based modelling

I tend to fit best in roles that need both technical depth and engineering taste: backend-heavy analytical systems, ML-systems-adjacent engineering, decision-support products, and full-stack delivery for complex domain workflows.
